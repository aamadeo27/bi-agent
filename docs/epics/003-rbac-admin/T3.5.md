# Permission Editor UI (S5) — tri-state schema tree

description: Build the S5 Permission Editor: two-panel layout, virtualized schema
tree (schema>table>column) with tri-state checkboxes (all/some/none), search,
detail panel with per-column read toggles, grant-all/revoke-all, staged "Unsaved
changes" badge, batch Save (PUT grants), cancel-with-confirm. Implement the WAI-ARIA
tree pattern (roles, arrow-key navigation, space toggles).

tag: frontend
deps: T3.4

acceptance criteria:
- Tree renders large schemas (virtualized) from the schema-tree endpoint.
- Tri-state behaves per UI/UX §7 (table checks columns; columns individually
  un-checkable → indeterminate parent).
- Save sends the batch grant set; cancel confirms when there are staged changes.
- ARIA tree pattern + keyboard nav verified; RTL + axe tests pass.
