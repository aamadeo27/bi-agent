# User↔role assignment API

description: Implement user management endpoints: `GET/PATCH /api/admin/users[/:id]`
to list users (name, email, role, status), assign a single role (or "No role"), and
suspend/reinstate. Tenant-scoped, admin-gated. (Invite creation lives in T2.4;
this task wires role assignment + status.)

tag: backend
deps: T3.1

acceptance criteria:
- List returns users with current role + status, tenant-scoped.
- Assign role (single role per user, v1) and suspend/reinstate persist correctly.
- Assigning a role takes effect on the user's next token refresh (GAP-17) — verified.
- Tests: assignment, suspend/reinstate, admin-gating, tenant isolation.
