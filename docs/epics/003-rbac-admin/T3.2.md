# Resource-grant API & schema-tree endpoint

description: Implement the schema/table/column grant model: `GET/PUT
/api/admin/roles/:id/grants` (read + batch-replace the role's grant set, matching
S5 "Save changes"), and `GET /api/admin/schema/:dataSourceId` returning the
schema>table>column tree with column types for the editor. Persist column-level
un-grants under a granted table as explicit grant rows (tri-state semantics).

tag: backend
deps: T3.1

acceptance criteria:
- `PUT grants` replaces the role's grant set atomically (batch); read returns it.
- Schema-tree endpoint returns schema/table/column + types for a data source.
- Tri-state correctly represented: table grant minus specific columns persists as
  explicit column grant rows (rbac-grant-model).
- Tests cover grant replace, tri-state edge cases, admin-gating, tenant isolation.
