# Role CRUD API & capability flags

description: Implement role management endpoints: `GET/POST/PATCH/DELETE
/api/admin/roles[/:id]`, tenant-scoped and admin-gated. Enforce unique role name
per tenant (≤64), optional description (≤256), and the `capabilities.canInspectQuery`
boolean (default false; seed Admin role true). Deleting a role surfaces the count of
affected users.

tag: backend
deps: T2.1

acceptance criteria:
- CRUD works, tenant-scoped; duplicate name → validation error.
- `canInspectQuery` defaults false; persisted and returned per Role contract.
- Delete reports affected-user count; only admin-capable roles may call these.
- Tests: CRUD, uniqueness, admin-gating, tenant isolation.
