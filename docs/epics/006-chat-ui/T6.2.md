# Streaming message render & timeline

description: Implement the chat timeline + message components: user MessageBubble,
SystemMessageBubble (markdown, streaming cursor), StreamingIndicator, auto-scroll
with "scroll to bottom" FAB. Consume the SSE client: render `token` deltas live,
swap the indicator on first token, and on `result` mount the ChartCard, on `block`/
`error`/`clarification` render the matching state component. Cache each message's
result envelope in TanStack Query (backs toggle/export — GAP-13).

tag: frontend
deps: T6.1, T5.5

acceptance criteria:
- Text streams token-by-token; indicator → bubble on first token; terminal handled.
- `result` mounts ChartCard; `block`/`error`/`clarification` render their components.
- Each message's envelope is cached client-side keyed by message id (no re-query on
  toggle/export).
- RTL test with a mocked SSE stream; axe passes on the timeline.
