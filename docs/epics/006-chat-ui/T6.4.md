# ChartCard: toggle + export (client cache)

description: Build the ChartCard wrapper: header (read-only chart-type badge +
toggle button + export button), chart/table body using the T6.3 components, loading/
empty/large-result banners. The chart↔table toggle reads the cached envelope (no
re-query — GAP-13) and remembers per-message view state. Export popover: PNG/JPEG
(client-side canvas render of the chart) and CSV/JSON (client-side serialization of
the cached rows), with the GAP-14 large-export warning above the threshold.

tag: frontend
deps: T6.3

acceptance criteria:
- Toggle switches chart↔table instantly from cache; per-message state persists;
  toggle hidden when type is table; `aria-pressed` reflects mode (§9/§10).
- Export produces PNG/JPEG (canvas) and CSV/JSON (serialized) downloads with the
  spec filename pattern; large-export warning shown above the threshold.
- Export serializes only the already-authorized cached envelope (no fresh query).
- RTL tests for toggle + each export; axe passes.
