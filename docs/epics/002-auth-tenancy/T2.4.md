# Invite-based user provisioning

description: Implement tenant-admin email invites: `POST /api/admin/users/invite`
creates an `invited` user and emails a signed, expiring invite token;
`POST /api/auth/invite/accept` validates the token and lets the invitee set a
password or link SSO, moving the user to `active`. Email delivery behind a small
mailer port (console/stub adapter for dev).

tag: backend
deps: T2.2

acceptance criteria:
- Admin invite creates an `invited` user and dispatches an invite token.
- Accept with a valid token activates the user (password set or SSO linked).
- Expired/used/invalid token → rejected.
- Tests cover invite → accept → login round trip and token expiry.
