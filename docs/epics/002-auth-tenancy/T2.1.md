# Tenant-scope & auth middleware

description: Implement the edge middleware chain for `apps/api`: (1) auth middleware
that validates the JWT access token and attaches `{userId, tenantId, roleId}`;
(2) tenant-scope middleware that sets the request connection `search_path` to
`tenant_<id>` and rejects any request body referencing another tenant's ids. Pure,
well-tested; this is a security boundary.

tag: backend
deps: T1.2, T1.3

acceptance criteria:
- Valid token → request context populated; invalid/expired → `AUTH` error.
- All DB access in the request runs in the resolved tenant scope.
- A request that supplies a foreign tenant id in the body is rejected (`TENANT`).
- Tests cover the tenant-leakage and foreign-id cases (testing.md security set).
