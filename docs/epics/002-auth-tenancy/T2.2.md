# Email+password auth & token sessions

description: Implement password auth: argon2id hashing, `POST /api/auth/login`,
`POST /api/auth/refresh` (rotating refresh in an httpOnly Secure SameSite cookie),
`POST /api/auth/logout`. Issue short-lived (~15m) JWT access tokens carrying
`{sub, tenantId, roleId, exp}`. Token TTL realizes GAP-17.

tag: backend
deps: T2.1

acceptance criteria:
- Login verifies argon2id hash, returns access token + sets refresh cookie.
- Refresh rotates both tokens; logout revokes refresh.
- Access token claims match contracts auth-api; TTL ~15m.
- Tests: wrong password, expired access, refresh rotation, and that a role change
  is reflected after refresh (GAP-17).
