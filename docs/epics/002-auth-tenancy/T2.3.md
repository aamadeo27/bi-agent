# Per-tenant SSO / OIDC

description: Add optional per-tenant OIDC using `openid-client`. Store IdP config
per tenant. Implement `GET /api/auth/sso/:tenant/start` and `/callback` (auth-code
flow). On callback, map the OIDC subject to an existing (invited/active) tenant
user; reject unknown subjects (no auto-provisioning beyond invite). Issue the same
short-lived tokens as password auth.

tag: backend
deps: T2.2

acceptance criteria:
- A tenant with OIDC configured can complete the code flow and receive tokens.
- Unknown OIDC subject → rejected (must be invited first).
- Tenants without OIDC are unaffected (password path still works).
- Tests cover successful callback and unknown-subject rejection (mock IdP).
