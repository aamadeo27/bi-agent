# Ask orchestrator & SSE endpoint

description: Implement the ask orchestrator and `POST
/api/conversations/:id/messages` SSE endpoint wiring the full Action A pipeline:
resolve identity/tenant/role → build LLM context (**schema metadata only, no row
data** — GAP-18) with windowed history → `generateQuery` → **permission gate (T5.2)**
→ validator (T5.3) → execute via the Query Proxy (T4.4) → stream `meta`/`token`/
`result` (or `block`/`error`) events → run chart selection (T5.4) → persist the
message + emit an audit event. Abort the upstream stream on client disconnect.

tag: backend
deps: T5.1, T5.2, T5.3, T5.4, T5.6, T4.4

acceptance criteria:
- End-to-end pipeline runs in order; gate precedes execution; block → `event: block`
  with PermissionBlock, nothing executed (GAP-12).
- SSE emits `meta`/`token`/`result`/`block`/`error`/`done` per chat-api; terminal
  event always sent; client disconnect aborts the LLM stream (no leak).
- Result wrapped in ResultEnvelope with auto-selected chartType; prompt context
  contains schema metadata only (asserted).
- Integration tests (mock provider, Testcontainers source) cover success, block,
  clarification, validation-fail, data-source-error, and follow-up (gate re-runs).
