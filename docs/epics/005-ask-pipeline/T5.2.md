# Permission gate (L1, pure) — block + explain

description: Implement the pure `evaluateGate({query, grants, dialect})` function:
parse the generated SQL with `node-sql-parser` (and the REST connector's declared
schema for REST) to extract every referenced schema.table.column, resolve aliases/
CTEs/subqueries and unqualified columns, and diff against the role's grant set. On
any missing resource return `{allow:false, missing[]}` → the pipeline emits the
PermissionBlock. Fail closed on unresolvable names. This is THE security boundary.

tag: backend
deps: T1.3

acceptance criteria:
- Resources are derived from the query AST, never the model's self-report.
- Any missing schema/table/column blocks the WHOLE query (no subsetting) — GAP-12;
  `missing[]` is exact and matches the PermissionBlock contract.
- Fails closed on unresolved columns/aliases; handles CTEs, subqueries, joins.
- Adversarial tests per testing.md security set: allow on fully-granted, block on
  any gap, never partial, deterministic.
