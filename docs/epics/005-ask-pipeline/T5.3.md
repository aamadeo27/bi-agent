# Query validator & injection guard

description: Implement the validator that runs after the gate, before execution:
allow-list read-only verbs (SQL `SELECT` only / REST declared read endpoints),
reject multi-statement, comment-hidden statements, DDL/DML, `;` chains, file I/O
and non-allow-listed set-returning functions; enforce a row LIMIT cap and a max
query length; parameterize literals where applicable. Returns a `ValidatedQuery` or
a `VALIDATION` error.

tag: backend
deps: T5.1, T5.2

acceptance criteria:
- Only read-only verbs pass; everything in the reject list is blocked (NFR-SEC-3).
- Row cap + query-length cap enforced; literals parameterized where possible.
- REST queries validated against the connector's declared endpoint/field schema.
- Adversarial tests: multi-statement, comment-hidden, DDL/DML, oversized — all rejected.
