# Conversations & durable history

description: Implement the conversations module + endpoints: `GET/POST
/api/conversations`, `GET /api/conversations/:id/messages`, `DELETE
/api/conversations/:id`. Persist conversations + messages durably (GAP-4 resolved:
retained until the user deletes them or the 365-day auto-purge job removes them —
see T5.8), tenant + user scoped. Store a `createdAt` per conversation/message so the
purge job can find expired rows. Store enough per message to rebuild context for
follow-ups and to back the result envelope/inspect view. Provide the
windowed-history helper the orchestrator uses.

tag: backend
deps: T2.1

acceptance criteria:
- Conversation + message CRUD works, tenant + user scoped; titles derived from the
  first user message.
- History is durable across sessions; delete removes a conversation + its messages.
- `getHistoryWindow(conversationId, budget)` returns context within the token budget.
- Tests: scoping, durability, delete cascade, windowing.
