# Chart-type selection (server, pure)

description: Implement the pure `selectChartType(columns, rows, rowCount)` function
per UI/UX §9 mapping: 1 categorical dim + 1 measure → bar; 1 time dim + measure(s)
→ line; 1 dim (3–8 distinct, part-of-whole) + measure → pie/donut; >8 distinct →
bar; 2+ dims / unrecognized / 0 rows → table; >2000 rows → table (GAP-8). Infer
each column's role (dimension/measure/time). Output feeds the ResultEnvelope
`chartType` + column roles.

tag: backend
deps: T5.1, T5.2

acceptance criteria:
- Returns the correct chart type for each row in the UI/UX §9 mapping incl.
  tie-break (prefer line when a time column is present) and pie cardinality limit.
- Column roles (dimension/measure/time) inferred from types/values.
- >2000 rows and 0 rows → table with the appropriate `notes`/`truncated`.
- Unit tests cover every mapping row + edge cases.
