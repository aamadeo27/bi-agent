# Query-inspect endpoint (capability-gated)

description: Implement `GET /api/messages/:id/query` returning the
GeneratedQueryView (query text, type, data-source name, executedAt, rowCount).
Authorize via the requester's role `capabilities.canInspectQuery` (GAP-1 locked
per-role toggle); return 403 (`AUTH`) otherwise. Tenant-scoped.

tag: backend
deps: T5.5

acceptance criteria:
- Returns GeneratedQueryView for a message the user may inspect.
- Denied (403/`AUTH`) when the role lacks `canInspectQuery`.
- Tenant-scoped; cannot inspect another tenant's/forbidden message.
- Tests: allowed role, denied role, tenant isolation.
