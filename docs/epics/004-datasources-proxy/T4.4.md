# Query Proxy with restricted-credential resolution (L2)

description: Implement the Query Proxy: the single component that executes
data-source queries. Given `(tenantId, roleId, dataSourceId, validatedQuery)`,
resolve the per-role credential from the vault (decrypt in memory), pick the right
connector, execute with timeout + row cap, and return the normalized result. The
credential is least-privilege at the source (per-role DB role / REST token +
allow-list) so it cannot exceed the role's grants — the L2 backstop to the gate.

tag: backend
deps: T4.1, T4.2, T4.3

acceptance criteria:
- Proxy resolves the `(tenant,role,source)` credential and never logs it.
- Execution applies timeout + row cap and returns normalized rows.
- Integration test (least-privilege Postgres role): a query touching a non-granted
  table is rejected at the source even if it reached the proxy (L2, NFR-SEC-1).
- Proxy is the only module that touches credentials (enforced + tested).
- Proxy executes via **raw drivers** (`pg`/`mysql2`/`@google-cloud/bigquery`/`undici`)
  on connections keyed by the restricted credential — **never via Prisma** (Prisma is
  control-plane only; using it here would bypass L2). Asserted/lint-enforced.
