# SQL connectors (PostgreSQL, MySQL, BigQuery)

description: Implement PostgreSQL, MySQL, and BigQuery adapters that **conform to the
existing `Connector` contract** (`datasource-connector`, frozen by T4.3 — see
`apps/api/src/datasource/connector.ts`). Define a SQL query-input type (e.g. `SqlQuery`)
and implement `Connector<SqlQuery>` for each adapter using `pg`, `mysql2`, and
`@google-cloud/bigquery`. Each adapter: connect with a supplied credential, run a
validated read-only query with a statement timeout + row cap, introspect schema
(schema > table > column + types for the schema-tree endpoint), and normalize
rows/columns into the `QueryResult` shape (column type + role inference). Connection
pooling/lifecycle is internal to the adapter.

**Do NOT define or modify the `Connector` interface.** It is owned by the
`datasource-connector` contract and lives in `apps/api/src/datasource/connector.ts`.
Implement against it — do not edit that interface block or its imports/exports.

tag: backend
deps: T4.1

implementation notes:
- Define a SQL query-input type (e.g. `SqlQuery`) and implement `Connector<SqlQuery>`.
- Do NOT edit the `Connector` interface block, its `QueryResult`/`QueryColumn`/`RestQuery`
  types, or the imports/exports in `apps/api/src/datasource/connector.ts`.
- Add adapter source file(s) under `apps/api/src/datasource/` (e.g. `pg-connector.ts`,
  `mysql-connector.ts`, `bigquery-connector.ts`) and **APPEND only their exports** to the
  barrel `apps/api/src/datasource/index.ts`.

acceptance criteria:
- Each adapter implements `Connector<SqlQuery>`:
  - `testConnection(): Promise<void>` — throw `ConnectorDataSourceError` on failure.
  - `introspect(dataSourceId): Promise<SchemaTree>` — map SQL schema > table > column
    (with types) into `SchemaTree`. Feeds T3.2's schema-tree endpoint.
  - `query(q: SqlQuery): Promise<QueryResult>` — run with statement timeout + row cap;
    normalize into `QueryResult` columns/rows (type + role inference), set
    `rowCount`/`truncated`.
- Pooling/connection lifecycle is internal — no connect/disconnect on the interface.
- The `Connector` interface in `connector.ts` is unchanged; only adapter files are added
  and their exports appended to `index.ts`.
- Tests use Testcontainers (pg/mysql) and a mocked/recorded BigQuery client.
