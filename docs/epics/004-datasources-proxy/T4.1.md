# Data-source registry & encrypted credential vault

description: Implement the data-source registry (`GET/POST/PATCH/DELETE
/api/admin/data-sources[/:id]`, tenant-scoped, admin-gated) storing name, type, and
connection config. Implement the credential vault: envelope encryption (AES-256-GCM
data keys under a KMS master key) for per-(tenant,role,source) credentials.
Credentials are never returned in API responses or logs.

tag: backend
deps: T3.1

acceptance criteria:
- Data-source CRUD works per DataSource contract; credentials write-only via API.
- Vault encrypts/decrypts credentials; ciphertext at rest, plaintext only in memory.
- No endpoint or log ever emits a credential (test asserts redaction).
- Tests: CRUD, encryption round-trip, admin-gating, tenant isolation.
