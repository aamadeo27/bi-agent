# Audit persistence & event emission

description: Implement the audit module: persist AuditEvent records (per-tenant
schema) and a `recordAudit(event)` helper, plus a `GET /api/admin/audit` endpoint
with filtering (date range, event type, user, data source) + pagination. Wire emit
calls at the security-relevant points: query executed/blocked/validation-failed
(ask pipeline), export (where applicable), role/permission changes, user-role
assignment, data-source changes, login/login_failed. `detail` must never contain
queried row data or credentials.

tag: backend
deps: T2.1, T3.1, T5.5

acceptance criteria:
- AuditEvent persisted per contract, tenant-scoped; `detail` carries no row data or
  secrets (asserted).
- Emit points fire for all listed event types (query_blocked records `missing[]`).
- `GET /api/admin/audit` filters + paginates, admin-gated.
- Tests: emission on block/execute/role-change/login, filtering, tenant isolation,
  redaction.
