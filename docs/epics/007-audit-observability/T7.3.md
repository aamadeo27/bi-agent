# Observability instrumentation (OTel + pino)

description: Wire OpenTelemetry traces + metrics and pino structured logging across
the API per monitoring-direction.md: span the ask pipeline (generation, gate
decision allow/block, validation, data-source exec, streaming), emit metrics for
gate decisions, end-to-end + data-source + LLM latency, LLM tokens/cost per
request and per tenant, the error-code taxonomy, throughput/error rate, and
streaming health (active/aborted streams). Tag every span/log with tenant/user/role/
requestId; never emit secrets or row data.

tag: infra
deps: T5.5

acceptance criteria:
- All monitoring-direction signals are emitted (gate, latency, tokens/cost, error
  taxonomy, throughput, streaming health).
- Spans/logs carry tenant/user/role/requestId; no credentials/tokens/row data
  (asserted by a redaction test).
- OTel exporter is config-driven so the `monitor` agent can point it at a backend.
- A smoke test asserts a gate-block produces the expected metric + span attributes.
